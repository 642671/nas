# 14. iSCSI Manager 的使用还有原理
> iSCSI 相关对象通常分成 3 层：
```
1.存储池/卷（Storage Pool / Volume）
    

- 这是后端真实的存储资源（RAID/TRAID 组成的空间）。
    

2.LUN（iSCSI LUN）= 虚拟磁盘本体
    

- 在 TOS 里创建的 LUN，本质是“从存储池里切出来的一块块块设备空间”。
    
- 这就是所谓的“虚拟磁盘”（对外表现为一块硬盘）。
    

3.Target（iSCSI Target）= 对外服务入口
    

- Target 有一个唯一标识IQN。客户端连接时就是连 Target。
```

# 15. S.M.A.R.T. 测试输出报告.
> 这个不常用，仅作为了解
![[{C71306FB-EAE1-4ECF-970E-5044CAAB57B0} 1.png]]
```
表头每一列是什么意思

-ID：SMART 属性编号（行业通用编号，但有些编号会被厂商“私用/重定义”）
    
-Attribute：属性名称（有些显示 Unknown_Attribute 表示该软件没识别出来）
    
-Value（当前值）：归一化健康分，通常越大越好（100/200 常见）
    
-Worst（最差值）：历史上出现过的最低（最差）归一化值
    
-Threshold（阈值）：低于此值通常判定该项“失败”（但 SSD 很多项并不会严格用阈值来判故障）
    
-Raw data（原始数据）：计数器/温度/次数等原始读数（最值得关注的一列）
```
**Raw data（原始值）最重要**，因为它更接近“真实计数/真实读数”。但关键点是——**Raw data 不是一个统一单位**：不同 ID 代表不同含义（次数、温度、扇区数、小时数、百分比等），而且 SSD 上有不少是厂商私有计数器。

# 19. 怎么完成存储池中的挂载
> 只有在存在损坏盘的情况下可以挂载
	- **挂载**：  
     • 在 **损坏存储池** 中，挂载可将在损坏阵列中的磁盘重新挂载并将存储池恢复
     为可用状态，无需重启系统。  
     • 在 **未挂载的存储池** 中，挂载可将在另一台设备上创建的存储池挂载并恢复
     为可用状态，无需重启系统。
