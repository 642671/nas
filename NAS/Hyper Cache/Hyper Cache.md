# Hyper Cache

Hyper Cache 是铁威马开发的一种 SSD 缓存加速技术。与传统的 SSD 缓存相比，Hyper Cache 能提供 2 种缓存模式供选择以适应不同的使用需求，还能为 SSD 创建磁盘阵列以增加缓存速度与安全性。铁威马 Hyper Cache 可成倍提升存储性能，为您打造更高效可靠的混合型存储。

  

## 管理与操作

通过功能配置的按钮，你可以进行以下操作：

- **创建**：创建一个 SSD 缓存。
- **编辑**：当 SSD 缓存阵列中的盘出现故障时，可用新的 SSD 进行替换修复。
- **删除**：删除已创建的 SSD 缓存。
## 如何创建 SSD 缓存？

1. 将 SSD 插入 TNAS 硬盘插槽。  
2. 前往 存储管理 > Hyper Cache，点击 "创建"。  
3. 选择一个卷用以创建缓存。  
4. 选择用于创建缓存的 SSD 并为其选择阵列类型，点击 "下一步"。  
5. 系统会提示您 SSD 中的所有数据将被删除以创建缓存，请点击 "确认" 以继续。输入管理员密码并再次确认以完成此步骤。  
6. 根据您的需要选择合适的缓存模式，点击 "下一步"。  
7. 在弹出的确认设置页面，点击 "确认"。

  

## 不同缓存模式的区别？

**平衡模式**： 此模式提供预加载的读缓存加速。数据将被同时写入 SSD 缓存与硬盘，数据的写速度将会一定程度的降低，但可以提升读的速度。平衡模式可以避免 SSD 出现故障，或者中途停电导致的丢失的风险；在该模式下，通过使用 2 块 SSD 组建 RAID 0 阵列可以提升缓存的速度。平衡模式适合对写缓存性能要求不高，但对读缓存性能要求较高的用户。

**读-写模式**： 此模式提供读与写两种缓存加速。数据将被先写入到 SSD 缓存，稍后再写入硬盘；读-写模式的缺点是不够安全，一旦 SSD 出现故障，或者中途停电，数据有可能会丢失；如果需要提升读-写模式的安全性，可以使用多块 SSD 组成 RAID 1 或者 RAID 5 阵列以为缓存 SSD 提供冗余，这样不仅可以增加缓存的容量，而且可以避免 SSD 故障导致的数据丢失风险。读-写模式适合对缓存读写性能要求较高的用户。
