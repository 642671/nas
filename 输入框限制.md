# 数字输入框要求
## 1. 卷（卷容量：__ GB）
### 1.1 创建卷容量配置要求
#### 1) 范围限制（业务校验）

- 最小值：**10 GB**
    - 合法示例：`10`、`10.0`、`10.00` 
- 最大值：**实际存储池可分配容量**（显示/计算得到的 max 值）
    - 合法示例：`max`、`max.0`、`max.00`（若max是整数）
> 判定规则：输入值 **必须满足** `10 ≤ value ≤ max`
#### 2) 格式限制（前端输入/提交校验）
**2.1 禁止前导 0（多位数）**
- 非法：`010`、`00010`、`0010.0`
**2.2 禁止“有小数点但没有小数”**
- 非法：`10.`、`0010.`、`0.`
**2.3 小数位数仅允许 0~2 位**
- 非法：`100.233`
**2.4 仅允许数字 + 1 个小数点**
- 非法：`10..`、 `10`、`10` 、`10,5`、`1e3`、`10GB`、中文字符、逗号分隔、其他特殊符号

### 1.2 编辑卷容量配置要求
#### 1) 范围限制（业务校验）
- **最小值 = 当前卷容量 current_capacity**
    - 合法示例：`current`、`current.0`、`current.00`（如果允许小数）
- **最大值 = 系统计算的可扩容上限 max_capacity**
    - 通常来源：存储池可分配容量（UI 展示的 Max 值/提示范围为准）
    - 合法示例：`max`、`max.0`、`max.00`
> 判定公式：`current_capacity ≤ value ≤ max_capacity`
#### 2) 格式限制（前端输入/提交校验）
**2.1 禁止前导 0（多位数）**
- 非法：`010`、`00010`、`0010.0`
**2.2 禁止“有小数点但没有小数”**
- 非法：`10.`、`0010.`、`0.`
**2.3 小数位数仅允许 0~2 位**
- 非法：`100.233`
**2.4 仅允许数字 + 1 个小数点**
- 非法：`10..`、 `10`、`10` 、`10,5`、`1e3`、`10GB`、中文字符、逗号分隔、其他特殊符号

## 2. 容量通知输入框（_ _ %）
> 包括两个输入框：卷、USB设备
#### 2.1 前置条件
前置条件：
	勾选选项‘启用容量不足通知’
在不勾选的情况下，输入框置灰，无法输入数字
#### 2.2 配置要求（卷/USB设备）
#### 1) 范围限制（业务校验）
- 最小值：**1**
- 最大值：**99**（创建卷的时候会默认有使用一定空间，所以选择最大值为99）
> 判定规则：输入值 **必须满足** `1 ≤ value ≤ 99`
#### 2) 格式限制（前端输入/提交校验）
**2.1 禁止前导 0（多位数）**
- 非法：`010`、`00010`
**2.2 禁止仅数字以外的任何格式**(小数点、字符、字母、中文等及其与数字任意组合都禁止)
- 非法：`+10、-1、1e3、10.0`
**2.3 长度限制为最大两位
- 非法：`100、10086`
