# 输入框相关限制
## 1. 卷输入框（卷容量：__ GB）
### 1.1 创建卷容量配置要求
#### 1) 范围限制（业务校验）

- 最小值：**10 GB**
    - 合法示例：`10`、`10.0`、`10.00` 
- 最大值：**实际存储池可分配容量**（显示/计算得到的 max 值）
    - 合法示例：`max`、`max.0`、`max.00`（若max是整数）
> 判定规则：输入值 **必须满足** `10 ≤ value ≤ max`
#### 2) 格式限制（前端输入/提交校验）
**2.1 禁止前导 0（多位数）**
- 非法：`010`、`00010`、`0010.0`
**2.2 禁止“有小数点但没有小数”**
- 非法：`10.`、`0010.`、`0.`
**2.3 小数位数仅允许 0~2 位**
- 非法：`100.233`
**2.4 仅允许数字 + 1 个小数点**
- 非法：`10..`、 `10`、`10` 、`10,5`、`1e3`、`10GB`、中文字符、逗号分隔、其他特殊符号

### 1.2 编辑卷容量配置要求
#### 1) 范围限制（业务校验）
- **最小值 = 当前卷容量 current_capacity**
    - 合法示例：`current`、`current.0`、`current.00`（如果current是整数）
- **最大值 = 系统计算的可扩容上限 max_capacity**
    - 通常来源：存储池可分配容量（显示/计算得到的 max 值）
    - 合法示例：`max`、`max.0`、`max.00`（若max是整数）
> 判定公式：`current_capacity ≤ value ≤ max_capacity`
#### 2) 格式限制（前端输入/提交校验）
**2.1 禁止前导 0（多位数）**
- 非法：`010`、`00010`、`0010.0`
**2.2 禁止“有小数点但没有小数”**
- 非法：`10.`、`0010.`、`0.`
**2.3 小数位数仅允许 0~2 位**
- 非法：`100.233`
**2.4 仅允许数字 + 1 个小数点**
- 非法：`10..`、 `10`、`10` 、`10,5`、`1e3`、`10GB`、中文字符、逗号分隔、其他特殊符号

## 2. 容量通知输入框（_ _ %）
> 包括两个输入框：卷、USB设备
#### 2.1 前置条件
前置条件：
	勾选选项‘启用容量不足通知’
在不勾选的情况下，输入框置灰，无法输入数字
#### 2.2 配置要求（卷/USB设备）
##### 1) 范围限制（业务校验）
- 最小值：**1**
- 最大值：**99**（创建卷的时候会默认有使用一定空间，所以选择最大值为99）
> 判定规则：输入值 **必须满足** `1 ≤ value ≤ 99`
##### 2) 格式限制（前端输入/提交校验）
**2.1 禁止前导 0（多位数）**
- 非法：`010`、`00010`
**2.2 禁止仅数字以外的任何格式**(小数点、字符、字母、中文等及其与数字任意组合都禁止)
- 非法：`+10、-1、1e3、10.0`
**2.3 长度限制为最大两位
- 非法：`100、10086`

## 3. 存储池自定义同步速度输入框
#### 3.1 前置条件
前置条件：
	在阵列同步速度的几种方式中勾选‘自定义’
在不勾选的情况下，输入框置灰，无法输入数字
### 3.2 阵列同步速度（自定义）配置要求

##### **1. 范围限制（业务校验）**
- 取值范围：**1 ~ 10000 MB/s**（合法示例：`1`、`1.0`、`1.23`、`10000`、`10000.00`）
- 判定规则：最小速度与最大速度均必须满足 **`1 ≤ value ≤ 10000`**
##### **2. 格式限制（前端输入/提交校验）**  
**2.1 仅允许数字 + 1 个小数点**
非法：`10..`、空格、逗号、`1e3`、中文、`10MB/s`、`10,5`（逗号小数）、其他特殊符号  
**2.2 禁止“有小数点但没有小数”**
非法：`10.`、`0.`  
**2.3 小数位数仅允许 0~2 位**
非法：`100.233`
##### **3. 字段关联限制（业务校验）**
- 约束关系：**最大速度 ≥ 最小速度**（允许相等）
- 判定规则：必须满足 **`max_speed ≥ min_speed`**（非法示例：最小速度 `100`、最大速度 `99.99`）

## 4. 新增S.M.A.R.T.测试输入框
#### 4.1 前置条件
前置条件：
	在新增S.M.A.R.T.中勾选‘启用计划任务’
在不勾选的情况下，输入框置灰，无法输入数字
### 4.2 新增 S.M.A.R.T. 计划任务-时间输入框限制

##### **1. 格式限制（前端输入/提交校验）**
- 输入格式：**HH:mm（24 小时制）**
	合法示例：`00:00`、`10:56`、`23:59`
- 仅允许：数字 `0-9` + **一个冒号 `:`**（不允许中文、空格、其他符号等及其任意组合）
- 不允许秒：`HH:mm:ss`（如 `10:56:30`）
- 不允许缺失位数：`10:6`、`1:06`（应要求两位小时+两位分钟，或自动补零后再校验）
##### **2. 范围限制（业务校验）**
- 小时范围：**00~23**（非法：`24:00`、`99:00`）
- 分钟范围：**00~59**（非法：`10:60`、`10:99`）
涉及到返回值的：
	如果正确输入则设置成功
	如果输入的格式错误，则恢复到上次设计的时间

## 6. 挂载虚拟磁盘输入框
### 6.1 **iSCSI Target IP输入框**  
##### **1.1 格式限制（前端输入/提交校验）**
	仅允许输入 **IPv4 地址**（点分十进制），示例：`10.18.13.45`
	仅允许数字 `0-9` + `.`（不允许空格、中文、字母、`:`、`/` 等及其任意组合）
	禁止连续点与结尾点（非法：`10..18.1.1`、`10.18.1.1.`）
	禁止段缺失（非法：`10.18.1`、`10.18`）
##### **1.2 范围限制（业务校验）**
	每段必须 `0~255`（非法：`256.1.1.1`、`999.0.0.1`）
	段数必须为 4 段（非法：`1.1.1`、`1.1.1.1.1`）
##### **1.3 前导 0 处理**
	建议禁止多位前导 0（非法：`010.018.013.045`）
	或允许但提交时归一化为无前导 0（如 `010.018.013.045` → `10.18.13.45`，二选一需产品定）
**补充：目前的情况是加上前导0会花费超级长的时间'90秒'（正常8秒）连接失败，目前的最简单方法是将带有前导0的默认为错**
	还有是否考虑ipv6提供虚拟磁盘等

### 6.2 **端口输入框**  
##### **1.1 输入类型**
	数字输入框，仅允许数字 `0-9`（不允许小数、负号、空格、中文、字母）。  
##### **1.2 范围限制（业务校验）**
	`1~65535`（常见 iSCSI 默认端口 `3260`）；非法：`0`、`65536`、`-1`、`1.5`。  
##### **1.3 前导 0（建议）**
	禁止多位前导 0（非法：`03260`）
**补充：目前的情况是前导数可以被归一化识别即（`03260`→`3260`）**  

### 6.3 **启用 CHAP 认证用户名与密码输入框**
#### 1.1 前置条件
前置条件：
	在挂载虚拟磁盘界面勾选‘启用 CHAP 认证’
在不勾选的情况下，输入框置灰，无法输入数字
两个输入框与创建虚拟磁盘的应用的两个输入框一致

用户名允许字符集（建议：字母数字及常见符号），密码需限制长度与不可见字符（不允许换行/空格首尾），并支持显示/隐藏。
